### Этап 5: КРИТИЧЕСКАЯ ПРОБЛЕМА — Cloudinary Signed Upload

**Проблема:**
При загрузке фото Cloudinary возвращал ошибку:
```
401 Unauthorized
Error: Invalid Signature
```

**Длительность проблемы:** 1-2 дня мучений

**Что пробовали:**
1. ✅ Проверили API Secret (27 символов, без пробелов)
2. ✅ Проверили таблицу `app_secrets`
3. ✅ Добавили `pgcrypto` в Supabase
4. ❌ Использовали `hmac()` — НЕ РАБОТАЛО
5. ❌ Добавляли `api_key` в строку подписи — НЕ РАБОТАЛО
6. ❌ Меняли порядок параметров — НЕ РАБОТАЛО
7. ❌ Чистили кэш GitHub Pages — НЕ ПОМОГЛО

**Почему не работало:**
Мы использовали **НЕПРАВИЛЬНЫЙ АЛГОРИТМ** генерации подписи!

Cloudinary **НЕ использует HMAC-SHA1**. Он использует **обычный SHA-1**, где секрет просто приклеивается в конец строки.

**Внешний консультант (ключевой момент!):**

Эрнест попросил сформулировать вопрос для внешнего консультанта. Вот что мы узнали:

**Ответ консультанта:**
> "Ошибка 401 Invalid Signature — классическая проблема. Cloudinary НЕ использует классический HMAC-SHA1.
> Он использует обычный хэш SHA-1, где секретный ключ просто конкатенируется (приклеивается) к концу
> сформированной строки параметров: строка_параметров + API_SECRET."

**Правильный алгоритм:**
```sql
-- НЕПРАВИЛЬНО (мы использовали):
v_signature := encode(hmac(v_string_to_sign, v_api_secret, 'sha1'), 'hex');

-- ПРАВИЛЬНО (как нужно):
v_signature := encode(
    digest(
        'folder=' || v_folder || '&timestamp=' || v_timestamp || v_api_secret,
        'sha1'
    ),
    'hex'
);
```

**Важные правила Cloudinary:**
1. Параметры сортируются **строго по алфавиту** (folder, timestamp)
2. `api_key` **НЕ входит** в строку подписи (передается отдельно в FormData)
3. **API_SECRET приклеивается** в конец строки БЕЗ знака `&`
4. Используется **обычный SHA-1**, а не HMAC

**Финальная функция `get_cloudinary_signature`:**
```sql
CREATE OR REPLACE FUNCTION get_cloudinary_signature(
    p_admin_id TEXT,
    p_object_id TEXT
)
RETURNS JSON
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
    v_photo_count INT;
    v_timestamp INT;
    v_signature TEXT;
    v_folder TEXT;
    v_api_secret TEXT;
    v_api_key TEXT;
    v_cloud_name TEXT;
    v_expected_admin_id TEXT := '2038206387';
BEGIN
    IF p_admin_id IS NULL OR p_admin_id != v_expected_admin_id THEN
        RAISE EXCEPTION 'Доступ запрещен';
    END IF;

    SELECT key_value INTO v_api_secret FROM app_secrets
    WHERE key_name = 'cloudinary_api_secret';
    SELECT key_value INTO v_api_key FROM app_secrets
    WHERE key_name = 'cloudinary_api_key';
    SELECT key_value INTO v_cloud_name FROM app_secrets
    WHERE key_name = 'cloudinary_cloud_name';

    IF v_api_secret IS NULL OR v_api_key IS NULL THEN
        RAISE EXCEPTION 'Секрет или ключ не найден';
    END IF;

    SELECT COUNT(*) INTO v_photo_count FROM photos
    WHERE object_id = p_object_id;
    IF v_photo_count >= 15 THEN
        RAISE EXCEPTION 'Лимит 15 фото';
    END IF;

    v_timestamp := extract(epoch from now())::int;
    v_folder := 'properties/object_' || p_object_id;
   
    v_signature := encode(
        digest(
            'folder=' || v_folder || '&timestamp=' || v_timestamp || v_api_secret,
            'sha1'
        ),
        'hex'
    );

    RETURN json_build_object(
        'signature', v_signature,
        'timestamp', v_timestamp,
        'folder', v_folder,
        'api_key', v_api_key,
        'cloud_name', v_cloud_name
    );
END;
$$;
```

## 🔧 ТЕХНИЧЕСКИЕ РЕШЕНИЯ И УРОКИ

### 1. Cloudinary Signed Upload

**Урок:** Cloudinary НЕ использует HMAC!

**Неправильно:**
```javascript
const signature = crypto.createHmac('sha1', apiSecret)
    .update(stringToSign)
    .digest('hex');
```

**Правильно:**
```javascript
const signature = crypto
    .createHash('sha1')
    .update(stringToSign + apiSecret)
    .digest('hex');
```

**Правила строки подписи:**
1. Параметры по алфавиту: `folder`, `timestamp`
2. `api_key` НЕ входит в строку
3. Secret приклеивается БЕЗ `&`: `folder=...&timestamp=...SECRET`
4. Использовать `digest()`, не `hmac()`

### 2. Telegram WebApp Integration

**Урок:** Всегда проверяйте существование `window.Telegram`

**Неправильно:**
```javascript
const tg = window.Telegram.WebApp;
tg.expand();
```

**Правильно:**
```javascript
const tg = window.Telegram && window.Telegram.WebApp
    ? window.Telegram.WebApp
    : null;

if (tg) {
    tg.expand();
    tg.ready();
}
```

### 3. Типы данных PostgreSQL

**Урок:** Строго соблюдайте типы данных!

**Неправильно:**
```javascript

```


## 🐛 ОТЛАДКА И FAQ

### Частые ошибки

**❌ Invalid Signature**
```
Причина: Неправильный алгоритм подписи
Решение:
1. Использовать digest(), не hmac()
2. Строка: 'folder=...&timestamp=...SECRET' (без api_key)
3. Проверить pgcrypto (включено ли)
4. Проверить API_SECRET (без пробелов)
```

** Cannot read properties of undefined**
```
Причина: window.Telegram не существует
Решение: Добавить проверку:
if (window.Telegram && window.Telegram.WebApp) { ... }
```

**❌ 400 Bad Request**
```
Причина: Неправильный тип данных
Решение:
- images_gallery: '' (строка), не []
- floor_plans_images: '' (строка), не []
- URL разделять запятой: 'url1,url2,url3'
```

**❌ Спиннер на мобильном**
```
Причина: Inline Telegram SDK блокирует выполнение
Решение:
1. Убрать inline SDK из <head>
2. Использовать внешнюю загрузку: <script src="...">
3. Добавить ?v=VERSION к URL для очистки кэша
4. Упростить код админки
```

** Вкладка "Агент" пустая**
```
Причина:
1. Неправильное имя поля (user_id вместо owner_id)
2. Данные хранятся в таблице pages, не agent_data
3. Ошибка в функции loadAgentData()

Решение:
1. Исправить user_id → owner_id
2. Читать текст из pages по page_key = 'about'
3. Удалять HTML-теги при отображении
```

## 📝 ПРИЛОЖЕНИЕ: ПОЛНЫЙ СПИСОК ВСЕХ ПРОБЛЕМ И РЕШЕНИЙ

### Проблема 1: Invalid Signature (Cloudinary)
**Длительность:** 1-2 дня 
**Сложность:** 🔥🔥🔥 (критическая)

**Симптомы:**
```
POST https://api.cloudinary.com/v1_1/dljo2y10n/image/upload
401 Unauthorized
Error: Invalid Signature
```

**Что пробовали:**
1. ✅ Проверили API Secret (27 символов)
2. ✅ Проверили таблицу app_secrets
3. ✅ Добавили pgcrypto
4. ❌ hmac() — не работало
5. ❌ Добавление api_key в строку — не работало
6. ❌ Изменение порядка параметров — не работало

**Корень проблемы:**
Cloudinary НЕ использует HMAC-SHA1. Он использует обычный SHA-1 с приклеенным секретом.

**Решение:**
```sql
v_signature := encode(
    digest(
        'folder=' || v_folder || '&timestamp=' || v_timestamp || v_api_secret,
        'sha1'
    ),
    'hex'
);
```

**Урок:** Всегда читайте документацию API внимательно!

```markdown
Вместо ручной настройки используется **«Мастер установки» (installer.html)** — простой веб-интерфейс, который прячет всю техническую сложность «под капот».

**Путь агента (максимально короткий):**
1. Агент регистрирует 3 обязательных сервиса (Supabase, Cloudinary, Telegram BotFather).
2. Открывает `installer.html`.
3. Копирует и вставляет API-ключи в простую форму.
4. Нажимает одну кнопку «Развернуть».
5. Скрипт автоматически создает таблицы в БД, настраивает секреты и выдает готовые ссылки на приложения.

**Запрещено:** Предлагать агенту ручное редактирование файлов, написание SQL-скриптов или настройку GitHub Pages вручную. Все это делает Мастер установки через API.
```
