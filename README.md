# 🏢 Real Estate Telegram Bot & Admin System

## 📖 Описание проекта
Полноценная экосистема для управления объектами недвижимости, работающая внутри Telegram. Система включает в себя Telegram Mini App для клиентов/агентов, мощную административную панель (v2) для управления контентом, облачное хранение изображений и надежный бэкенд на базе Supabase.

Проект оптимизирован для работы как внутри Telegram, так и в обычных мобильных/десктопных браузерах.

## 🚀 Ключевые возможности

### 📱 Для пользователей (Telegram Web App)
- Просмотр каталога объектов недвижимости
- Фильтрация и поиск (в разработке)
- Быстрый просмотр фото и планировок
- Интерфейс, адаптированный под мобильные устройства

### 🛠 Для администратора (Admin Panel v2)
- **CRUD операции:** Создание, чтение, обновление, удаление объектов
- **Медиа-менеджмент:** Загрузка главного фото, галереи и планировок
- **Безопасная загрузка:** Подписанные запросы (Signed Upload) в Cloudinary
- **Статусы:** Управление видимостью объектов (Активен/Скрыт)
- **Кроссплатформенность:** Работает в браузере и внутри Telegram

## ⚙️ Установка и Настройка

### 1. База данных (Supabase)
1. Создайте проект в Supabase.
2. Включите расширение `pgcrypto`: `Database → Extensions → pgcrypto → Enable`.
3. Создайте таблицу `app_secrets` для хранения ключей.
4. Вставьте ключи Cloudinary (API Key, API Secret, Cloud Name).
5. Создайте функцию `get_cloudinary_signature` (код в разделе "Технические решения").

### 2. Frontend (GitHub Pages)
1. Загрузите `index.html` в репозиторий.
2. В настройках репозитория включите GitHub Pages (Branch: `main`, Folder: `/(root)`).
3. Полученный URL используйте для привязки к Telegram Bot.

### 3. Telegram Bot
1. Создайте бота через `@BotFather`.
2. Используйте команду `/newapp` или настройте Menu Button.
3. Укажите URL вашей страницы GitHub Pages.

## 💡 Ключевые технические решения и уроки

В ходе разработки v2 были выявлены и исправлены критические архитектурные нюансы:

###  1. Исправление типов данных (PostgreSQL)
- **Проблема:** Ошибка `400 Bad Request` при создании объекта.
- **Причина:** Поля `images_gallery` и `floor_plans_images` в БД имеют тип `text`, а код отправлял массивы `[]`.
- **Решение:** Инициализация полей пустыми строками `''` и хранение URL через разделитель запятой.

###  2. Безопасная работа с Telegram API
- **Проблема:** Ошибка `Cannot read properties of undefined` при открытии в обычном браузере.
- **Решение:** Добавлена проверка существования объекта `window.Telegram`. Приложение теперь gracefully деградирует в браузере, сохраняя функционал.

### 🔹 3. Cloudinary Signed Upload (ГЛАВНОЕ ОТКРЫТИЕ)
- **Проблема:** Ошибка `401 Unauthorized - Invalid Signature`.
- **Причина:** Использование алгоритма `hmac()` вместо требуемого Cloudinary формата.
- **Правильный алгоритм:**
  1. Параметры сортируются по алфавиту (`folder`, `timestamp`).
  2. `api_key` **не включается** в строку подписи (передается отдельно в FormData).
  3. Используется обычный `digest('sha1')`, где **API Secret приклеивается в конец строки** без разделителей.
  ```sql
  -- Правильная генерация в PL/pgSQL:
  v_signature := encode(digest('folder=' || v_folder || '&timestamp=' || v_timestamp || v_api_secret, 'sha1'), 'hex');
  ```

## 🆘 Отладка и FAQ

**❓ Фото не загруются (Invalid Signature)**
- Убедитесь, что включено расширение `pgcrypto`.
- Проверьте, что `API_SECRET` в БД не содержит лишних пробелов или переносов строк.
- Убедитесь, что используется функция `digest()`, а не `hmac()`.

**❓ Приложение падает с ошибкой `Telegram is undefined`**
- Проверьте, что в коде есть проверка `if (window.Telegram && window.Telegram.WebApp)`.
- Убедитесь, что скрипт `telegram-web-app.js` загружается корректно.

**❓ Объект создается, но фото нет**
- Проверьте консоль браузера (F12) на ошибки CORS или Network Timeout.
- Убедитесь, что поля `images_gallery` передаются как строки `''`, а не массивы `[]`.
